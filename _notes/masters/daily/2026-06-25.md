ok it's training an encoder to map samples back to latent space
sample prior -> decode w/ trained model -> map back to latent -> gilbo loss
- where loss is `encoder(x).log_prob(z) - prior.log_prob(z)`
- so it basically measures how well the encoder is doing compared to the prior itself from which the `z` was drawn
- and this, allegedly, is a lower bound on mutual info `I(X, Z)`, which we only tighten by training
